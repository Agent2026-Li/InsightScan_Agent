## 🌟 核心业务场景

### 🧭 1. 售前/售后全域智能问答 (Agentic RAG Pipeline)

系统重构了传统 RAG 的呆板逻辑，将其升级为 **Agentic 检索增强架构**。当面对用户的非结构化高难咨询（如 *“扫地机激光雷达报错，家里有长毛金毛该配什么滤网”*）时，Agent 不会进行盲目的盲检，而是进入 **ReAct（思考→行动→观察）循环**，自主拆解意图，先路由至硬件说明书库，再叠加特殊耗材兼容度工具，给出多维度交叉的确定性解决方案。

### 📊 2. 个性化运行日志穿透与报告渲染 (Telemetry Gateway)

在用户授权前提下，系统可无缝对接扫地机器人底层的非结构化物理运行日志与清扫轨迹 CSV。系统内部的参数提取网关能精准捕捉自然语言中的 `User_ID` 与时间槽位，动态驱动 Pandas 运算引擎，自动分析覆盖率、重复率及耗材剩余量，一键渲染出图文并茂的 **《家庭清扫效率白皮书与空间优化指南》**。

### 🛡️ 3. 私有化容错防御网关 (Robust Parsing Gateway)

针对本地轻量化模型（如 Qwen3-1.7B / Gemma3-1B）在离线环境下进行 Tool Calling 容易发生 JSON 格式崩溃或混入 `<think>` 思考链的行业痛点，自研了一套**高鲁棒性正则表达式防御解析网关与 Pydantic 协议硬约束**，使轻量小模型在百元级边缘设备上的工具调用稳定性跃升，真正做到安全、隐私、不出本地。

## 🛠️ 系统技术架构

### 1. 具身智能中枢决策流

平台基于前后端解耦架构构建，控制流与数据流完全闭环：

```
【 物理感知/交互层 】   用户诉求 / 硬件日志 ──► Gradio 看板 / 智能控制台
                                                  │
                                                  ▼
【 智能体推理中枢 】   LangChain ReAct 动态状态机 (Thought -> Action -> Observe)
                                                  │
                ┌─────────────────────────────────┴─────────────────────────────────┐
                ▼ (产品原理/故障疑难)                                              ▼ (日志挖掘/效能评估)
【 敏捷工具矩阵 】  Agentic RAG 工具链                                               端到端报告生成网关
                └── ChromaDB 密集向量空间 (bge-m3)                                 └── Pandas 物理日志分析流
                                                  │
                                                  ▼
【 轻量推理内核 】   Ollama 私有化集群 (Qwen3 / DeepSeek-R1 / Gemma3)
                                                  │
                                                  ▼
【 最终交付层 】     防御性正则流结构化剥离 ──► 流式响应、可视化图表与性能预警

```

### 2. 核心技术栈矩阵

| 架构层级 | 选用组件 | 生产环境具体职责 |
| --- | --- | --- |
| **交互控制台** | `Gradio / FastAPI` | 构建低延迟、高并发的 Web 交互看板，完美支持双栏异步加载流 |
| **智能体骨架** | `LangChain 0.3.27` | 封装标准 ReAct 推理看板，提供长多轮对话上下文（ChatMemory）持久化 |
| **边缘推理集群** | `Ollama` | 统一接管本地多模型调度，支持在百元级硬件上实现离线流畅推理 |
| **向量存储引擎** | `ChromaDB 1.0.15` | 轻量化本地持久化向量库，支持对 10 大类扫地机 QA 数据进行语义召回 |
| **基座多模型** | `Qwen3 / DeepSeek / Gemma` | 支撑从 1B 对话、1.5B 深度思考到 8B 复杂推理的动态选型和降级容灾 |

## 📂 工业级项目模块结构

项目采用严谨的量产级微服务工程目录进行组织，告别玩具式单文件代码：

```
InsightScan/
├── 📁 server/             # 后端微服务：基于 FastAPI 搭建的 Agent API 网关与跨域路由
├── 📁 agent/              # 智能体中枢：ReAct 核心调度器、Prompt 提示词模板套件
├── 📁 tools/              # 敏捷工具箱：RAG 检索工具、Pandas 报表工具、正则解析流
├── 📁 storage/            # 数据存储：ChromaDB 向量空间根目录与 CSV 运行日志底表
├── 📁 client/             # 前端交付：Gradio 宽屏看版与高性能 CLI 终端双客户端
└── 📄 requirements.txt    # 环境配置：锁定全链路工程级依赖版本

```

## ⚡ 极简热启动

系统支持本地环境快速热启动，全链路模型自主接管：

```bash
# 1. 启动大模型边缘微服务 (确保 Ollama 已后台运行)
ollama run qwen3:8b

# 2. 一键拉起 FastAPI 后端 Agent 核心网关
python server.py

# 3. 启动现代化 Web 交互看板
python client/web_client.py

```

> 📌 **最低硬件要求**：
> * 支持 Windows/Linux 本地运行，建议显存 $\ge$ 6GB（或 CPU 内存 $\ge$ 16GB）。
> * 默认搭载的 `bge-m3` 及 `Qwen3` 均已完成内嵌量化校准，完美兼顾首字延迟与生成忠实度。
